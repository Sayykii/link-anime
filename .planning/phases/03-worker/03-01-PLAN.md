   - Use models.UpscaleStatus* constants for status values

   - UPDATE upscale_jobs SET status='pending', started_at=NULL WHERE id=? AND status='running'
   - Check RowsAffected - 0 means job already completed/failed (no reset needed)
   - This is atomic to prevent race between shutdown and job completion

  <name>Task 2: Create Worker struct with Start/Stop lifecycle</name>

  <action>
Create `internal/upscale/worker.go` following the DownloadMonitor pattern exactly:

```go
type Worker struct {
    engine     *Engine
    hub        *ws.Hub
    
    // Lifecycle channels (matches DownloadMonitor)
    stopCh     chan struct{}
    done       chan struct{}
    
    // Current job tracking for graceful shutdown
    currentJob int64
    cancelFunc context.CancelFunc
    mu         sync.Mutex
}
```

1. **NewWorker(hub *ws.Hub, shaderDir string) *Worker**
   - Create Engine with NewEngine(shaderDir)
   - Initialize channels: stopCh and done as make(chan struct{})
   - Return configured Worker

2. **Start()**
   - Spawn `go w.run()` (matches DownloadMonitor.Start)

3. **Stop()**
   - close(w.stopCh)
   - <-w.done (wait for run() to complete)
   - (matches DownloadMonitor.Stop)

4. **run()** (private)
   - defer close(w.done) at start
   - Create ticker: time.NewTicker(3 * time.Second) per WRK-02
   - defer ticker.Stop()
   - Initial poll: w.processNext()
   - for/select loop:
     - case <-w.stopCh: w.handleShutdown(); return
     - case <-ticker.C: w.processNext()

5. **processNext()** (private, stub for now)
   - Call database.GetNextPendingJob()
   - If nil, return (no job)
   - Log: `[upscale] found pending job %d: %s`
   - (Actual processing added in Plan 02)

6. **handleShutdown()** (private, stub for now)
   - Log: `[upscale] worker shutting down`
   - (Graceful shutdown added in Plan 02)

Import: context, log, sync, time, link-anime/internal/ws
